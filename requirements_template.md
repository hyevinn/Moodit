## 1. 프로젝트 개요

### 1.1 프로젝트 목표
> Moodit(무드잇)은 사용자의 소비 정보를 기반으로 소비 유형과 소비 성향을 분석하여 자신의 소비 습관을 직관적으로 이해할 수 있도록 돕는 AI 기반 소비 분석 서비스이다.
기존 가계부 앱의 단순 기록 기능을 넘어, 소비 이유와 소비 특징을 함께 분석하고 시각화하여 사용자가 자신의 소비 패턴을 쉽고 흥미롭게 파악할 수 있도록 하는 것을 목표로 한다.


### 1.2 대상 사용자
> 자신의 소비 습관과 소비 성향을 분석하고 싶은 대학생
> 감정 및 소비 이유 기반 소비 패턴을 확인하고 싶은 사용자


### 1.3 핵심 가치
> 소비 금액 기록뿐 아니라 소비 이유와 소비 유형을 함께 분석하여 AI 기반 소비 성향 리포트를 제공한다.

## 2. 기능 요구사항 (Functional Requirements)

> 시스템이 *무엇을 하는가*를 정의한다. 최소 5개 이상 작성
> 문장 패턴: "사용자는 ~할 수 있다" / "시스템은 ~한다"

### FR-01: [선택형 소비 입력]

- **설명**: 사용자는 선택형 UI를 통해 소비 정보를 간단하게 입력할 수 있다.
- **수용 조건**:
  - [ ] 사용자는 소비 카테고리를 선택할 수 있다.
  - [ ] 사용자는 소비 금액 범위를 선택할 수 있다.
  - [ ] 사용자는 소비 이유를 선택할 수 있다.

### FR-02: [소비 정보 분석 요청 기능]

- **설명**: 사용자는 입력한 소비 정보를 기반으로 소비 성향 분석을 요청할 수 있다.
- **수용 조건**:
  - [ ] 입력한 소비 정보가 분석 화면으로 전달된다.
  - [ ] 분석 요청 시 Gemini API가 호출된다.

### FR-03: [소비 성향 분석 기능]

- **설명**: 시스템은 사용자의 소비 정보를 기반으로 소비 유형과 소비 성향을 분석한다.
- **수용 조건**:
  - [ ] 소비 이유를 기반으로 기본 소비 유형을 분류한다.
  - [ ] Google Gemini API를 활용하여 소비 성향 리포트를 생성한다.

### FR-04: [소비 리포트 제공 기능]

- **설명**: 사용자는 AI가 생성한 소비 성향 리포트를 확인할 수 있다.
- **수용 조건**:
  - [ ] 분석 결과가 텍스트 형태로 출력된다.
  - [ ] 소비 특징 및 소비 성향이 포함된다.
  - [ ] 사용자는 소비 분석 결과를 화면에서 확인할 수 있다.

### FR-05: [소비 분석 결과 시각화 기능]

- **설명**: 시스템은 사용자가 입력한 소비 정보를 시각적으로 제공한다.
- **수용 조건**:
  - [ ] 선택한 소비 카테고리와 소비 이유를 카드 UI 형태로 표시한다.
  - [ ] 분석된 소비 유형 결과를 카드 UI 형태로 제공한다.
  - [ ] 소비 성향 키워드를 태그 형태로 제공한다.

### FR-06: [결과 화면 UI 기능]

- **설명**: 시스템은 소비 분석 결과를 직관적인 UI 형태로 제공한다.
- **수용 조건**:
  - [ ] Jetpack Compose 기반 UI가 적용된다.
  - [ ] 사용자는 하나의 화면에서 소비 분석 결과를 확인할 수 있다.

## 3. 비기능 요구사항 (Non-Functional Requirements)

> *어떻게 동작하는가*를 정의한다. 직접 보이지 않지만 품질을 결정한다.

### 3.1 성능 (Performance)

- 메인 화면 진입 후 5초 이내에 주요 UI를 표시한다.
- 소비 분석 결과는 요청 후 5초 이내에 출력한다.

### 3.2 보안 (Security)

- 사용자의 개인정보를 수집하지 않는다.
- Gemini API 호출 시 API Key가 외부에 노출되지 않도록 관리한다.

### 3.3 사용성 (Usability)

- 사용자는 복잡한 텍스트 입력 없이 선택형 UI만으로 소비 정보를 입력할 수 있어야 한다.
- 버튼 선택 시 색상이 변경되어 현재 선택 상태를 직관적으로 확인할 수 있어야 한다.
- 소비 분석 결과는 카드 UI와 자연어 리포트 형태로 이해하기 쉽게 제공되어야 한다.

### 3.4 호환성 (Compatibility)

- Android API 24 이상 환경에서 동작해야 한다.
- 다양한 Android 화면 크기에서 UI가 정상적으로 표시되어야 한다.

### 3.5 기타

- MVP 범위 내 핵심 기능 중심으로 구현한다.
- 유지보수가 쉽도록 기능별 모듈 구조를 적용한다.
