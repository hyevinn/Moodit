# Moodit 요구사항 정의서 (requirements_template.md)

## 1. 개요 (Introduction)

### 1.1 목적
본 문서는 AI 기반 소비 심리 분석 모바일 애플리케이션 'Moodit'의 기능적 및 비기능적 요구사항을 상세히 정의한다. 앱 개발 및 검증 기준으로서 동작 사양을 명시한다.

### 1.2 대상 독자
- 소비 이유와 금액을 기입하고 감정 기반 소비 유형을 파악하고자 하는 사용자
- AI 소비 성향 분석 결과를 확인하고 이를 바탕으로 행동을 개선하고 싶은 사용자

### 1.3 핵심 가치
- 단순 소비 금액의 조회를 넘어 감정적 소비 동기에 초점을 맞춘다.
- 하루 동안 기록한 다건의 소비를 누적 집계하여 입체적으로 분석한다.
- 시각적인 통계 차트와 AI가 생성한 개인화된 인사이트 조언을 제공한다.

## 2. 기능 요구사항 (Functional Requirements)

### FR-00: [위치 정보 수집 기능]
- **설명**: 사용자의 위치 정보를 수집한다.
- **수용 조건**:
  - [ ] 사용자의 위치 정보 수집 권한 허용 여부를 확인한다.
  - [ ] GPS 기반 위/경도를 수집하여 한글 주소명으로 변환 및 노출한다.

### FR-01: [알림 수신 동의 기능]
- **설명**: 사용자가 알림 수신에 동의하고 조건형 알림을 수신한다.
- **수용 조건**:
  - [ ] 사용자의 알림 권한 허용 여부를 확인하고 동의 버튼을 제공한다.
  - [ ] 특정 조건(총 금액이 10만원 이상이거나 늦은 22시 이후 소비 시 등)이 만족되면 사용자 알림을 발송한다.

### FR-02: [홈 화면 제공 기능]
- **설명**: 시스템은 서비스 소개 및 이전 분석 데이터 복원 화면을 제공한다.
- **수용 조건**:
  - [ ] 홈 화면에서 '새 소비 분석하기' 버튼을 클릭하면 소비 입력 화면으로 이동한다.
  - [ ] 상단 인사말 영역의 첫 번째 줄에 "안녕하세요 👋" 텍스트만 단독 노출하고, 두 번째 줄에 안내 문구와 캐릭터(`today_character`, `65.dp`)를 양 끝에 정렬 배치(`Arrangement.SpaceBetween`)한다.
  - [ ] 캐릭터가 아래로 내려오거나 인사이트 카드 위에 뜨는 느낌이 없도록, 인사말 수평 정렬(`verticalAlignment = Alignment.CenterVertically`) 및 하단 카드와의 여백(`20.dp` Spacer)을 확보한다.
  - [ ] 홈 화면 중앙의 '✨ 최근 소비 인사이트' 카드 내부에는 가독성을 극대화하기 위해 캐릭터를 제외하고 **인사이트 제목과 문구만 표시**하며, 글자 길이에 따라 세로 크기가 동적 증가(`heightIn(min = 140.dp)`)한다.
  - [ ] 최근 분석 결과 정보(소비 유형, 설명, 카테고리, 금액, 소비 이유)를 Preferences DataStore로부터 복원하여 화면에 실시간으로 표시한다.
  - [ ] 이전 분석 기록이 없는 초기 진입 시에는 기본 문구("오늘의 소비에는 오늘의 감정이 담겨 있어요.") 및 기본 아이콘을 노출한다.

### FR-03: [소비 기록 및 다건 내역 관리]
- **설명**: 사용자는 소비 내역을 누적 등록하고 수정/삭제하며 일일 내역을 임시 관리한다.
- **수용 조건**:
  - [ ] 소비 카테고리(식비, 카페, 교통, 쇼핑, 취미, 자기계발, 기타) 선택 기능을 제공한다.
  - [ ] 숫자로 된 소비 금액을 텍스트 필드를 통해 직접 입력할 수 있으며 숫자 키패드를 지원한다. 금액은 3자리마다 천 단위 쉼표(,) 포맷팅이 적용된다.
  - [ ] 카드형 버튼 UI를 통해 소비 이유(필요 소비, 자기만족, 기분전환, 유행 영향)를 선택할 수 있으며 카드 내부에 이모지와 텍스트를 함께 표시한다.
  - [ ] 오늘 소비 기분 한마디(메모)를 최대 100자까지 선택 입력할 수 있다.
  - [ ] 입력한 내역은 '오늘의 소비 내역' 목록 영역에 임시 저장(누적)되며, `LazyColumn`을 통해 출력된다.
  - [ ] 각 리스트 아이템에는 '수정' 및 '삭제' 버튼을 제공하여, 개별 내역의 수정 반영 및 목록에서 삭제 처리를 지원한다.
  - [ ] 저장된 내역들의 총 소비 금액 합계가 자동으로 합산 및 갱신되어 노출된다.

### FR-04: [소비 정보 분석 요청 기능]
- **설명**: 사용자는 입력한 소비 정보를 기반으로 소비 성향 분석을 요청할 수 있다.
- **수용 조건**:
  - [ ] 분석 시작을 위해서는 오늘의 소비 내역이 최소 1건 이상 등록되어 있어야 한다.
  - [ ] 누적 저장된 다건의 소비 내역 리스트 전체가 분석 및 로딩 화면으로 안전하게 전달된다.
  - [ ] 분석 요청 시 `LoadingScreen`에서 AI 분석이 수행 중임을 안내하는 로딩 애니메이션 상태를 표시한다.

### FR-05: [AI 소비 성향 분석 기능]
- **설명**: 시스템은 사용자의 다건 소비 내역을 종합적으로 분석해 한글 명사형 소비 유형과 가독성 높은 한 줄 조언을 생성한다.
- **수용 조건**:
  - [ ] GroqCloud API 및 Llama 3.3 LLM 모델을 활용해 카테고리별 누적 금액, 소비 이유, 빈도를 분석한다.
  - [ ] AI 분석 결과는 한자(Hanja), 일본어 및 불필요한 영어를 배제하고 자연스러운 한국어 문장으로 도출한다.
  - [ ] 분석된 [소비 유형]은 '필요형 소비자', '자기보상형 소비자' 등 명확한 한국어 명사로 표기한다.
  - [ ] 리포트로부터 추출되어 메인 화면에 쓰일 [한 줄 조언]은 아래의 조건들을 엄격히 적용해 딱 1문장으로 요약 및 생성한다.
    - **유기적 흐름**: 조언 사항을 나열하지 않고 유기적으로 이어지는 문장으로 구성.
    - **글자 수 범위**: 줄바꿈 시 어색하지 않도록 공백 포함 **40~60자 내외**의 길이 준수.
    - **부드러운 어미**: `~해 보세요`, `~이어가 보세요`, `~유지해 보세요`, `~어떨까요?`와 같이 친근하고 부드러운 종결 어미 사용.
    - **개인화 조언**: 실제 입력된 소비 내역에 부합하는 실질적인 행동 조언 제공.

### FR-06: [소비 리포트 제공 기능]
- **설명**: 사용자는 분석 리포트를 확인하고 뒤로가기 스택을 정리하며 홈 화면으로 돌아간다.
- **수용 조건**:
  - [ ] AI 리포트 텍스트 및 소비 특징/습관 정보가 결과 화면에 출력된다.
  - [ ] 결과 노출 즉시 최근 분석 결과 및 40~60자 인사이트 핵심 문장이 Preferences DataStore에 즉시 백업 저장된다.
  - [ ] 결과 화면 가장 하단에 '소비 분석하기' 버튼과 모서리 둥글기(18.dp), 높이(60.dp), 눌림 애니메이션이 완벽히 똑같은 **'메인화면으로 돌아가기'** 버튼이 배치된다.
  - [ ] '메인화면으로 돌아가기' 클릭 시 메인 화면(라우트 `"main"`)으로 이동하며, `popUpTo("main") { inclusive = true }` 설정을 사용해 분석 이전의 백스택(Result, Loading, Input 화면 등)을 비워내어 뒤로가기 시 이전 분석으로 재진입하는 일이 없도록 방지한다.

### FR-07: [결과 화면 시각화 기능]
- **설명**: 시스템은 누적 소비 분석 결과를 시각적으로 제공한다.
- **수용 조건**:
  - [ ] **소비 비율 파이 차트**: 카테고리별 누적 금액 비율을 계산해 `Canvas` 상에 파이 차트로 그리고 백분율 및 범례를 제공한다.
  - [ ] **소비 금액 막대 그래프**: 카테고리별 금액을 시각화해 주는 커스텀 막대 차트를 그리고 금액 텍스트를 함께 표기한다.

### FR-08: [결과 화면 통합 제공 기능]
- **설명**: 시스템은 결과 요약, 그래프, AI 리포트 등을 한 화면에 통합 구성한다.
- **수용 조건**:
  - [ ] 소비 정보 카드, 소비 유형 카드, 소비 성향 키워드 태그가 표시된다.
  - [ ] 시각화 차트 영역 및 상세 AI 리포트 텍스트 영역이 순서대로 배치된다.
  - [ ] 입력된 일일 메모들이 있을 경우 쉼표로 연결한 '오늘의 소비 한마디 모음' 카드를 함께 노출한다.

### FR-09: [테마 전환 기능]
- **설명**: 사용자는 화면 테마를 다크 모드 또는 라이트 모드로 전환할 수 있다.
- **수용 조건**:
  - [ ] 다크/라이트 모드 전환 기능 제공 및 전체 UI 색상이 실시간으로 변경된다.
  - [ ] 적용된 테마 설정 상태가 Preferences DataStore 등을 통해 앱 재실행 간에도 유지된다.

### FR-10: [브랜드 스플래시 화면 제공 기능]
- **설명**: 앱 최초 구동 시 브랜드 마스코트 캐릭터 스플래시를 노출하고 메인 테마로 부드럽게 화면을 이관한다.
- **수용 조건**:
  - [ ] `androidx.core:core-splashscreen` 라이브러리를 연동해 Android 12 이상 스플래시 화면을 설정한다.
  - [ ] `MainActivity.kt`에서 `installSplashScreen()`을 호출하여 스플래시 라이프사이클을 통제한다.
  - [ ] 스플래시 아이콘의 원형 마스크 잘림 현상을 방지하기 위해, 사방(특히 하단)에 충분한 안전 마진 여백이 포함된 전용 이미지 `@drawable/today_character_splash`를 적용한다.
  - [ ] 캐릭터 크기는 화면 중앙에서 약 60dp 수준으로 작게 정렬 배치되어 과도한 화면 점유나 부담스러움을 예방한다.
  - [ ] 스플래시 노출이 끝나면 지정된 메인 포스트 테마(`postSplashScreenTheme`)로 이송 전환한다.

## 3. 비기능 요구사항 (Non-Functional Requirements)

### 3.1 성능 (Performance)
- 앱 구동 후 3초 이내에 스플래시 전환이 끝나고 홈 화면이 표출되어야 한다.
- 소비 분석 API 응답은 네트워크가 원활할 시 5초 이내에 처리 및 완료되어야 한다.

### 3.2 보안 (Security)
- 사용자의 민감한 금융 및 개인정보를 서버나 로컬 데이터베이스에 저장하지 않는다.
- AI API 연동 자격 증명(Groq API Key)은 `local.properties`와 Gradle 컴파일 설정을 연동한 `BuildConfig` 필드로 빌드 타임에 관리하여 소스코드에 하드코딩되지 않도록 보안을 강화한다.

### 3.3 사용성 (Usability)
- 금액 직접 입력 폼은 천 단위 구분 기호(,)가 실시간 자동 포맷팅되며, 숫자 키패드가 원활히 전환되어야 한다.
- 소비 이유 카드 UI는 이모지가 포함되어 직관적이어야 하며 선택 시 시각적 피드백(강조 색상 및 테두리선 변환)이 주어져야 한다.
- 분석 결과 화면 하단 '메인화면으로 돌아가기' 버튼은 누를 때 축소 애니메이션을 주어 물리적 누름 효과를 제공해야 한다.
- 로컬 저장소 캐싱을 지원해 사용자가 앱 재실행 시에도 이전 최종 소비 분석 데이터를 별도의 네트워크 요청 없이 즉시 열람할 수 있도록 한다.

### 3.4 호환성 (Compatibility)
- Android SDK 26 (minSdk) 이상 및 SDK 35 (targetSdk) 이하 범위에서 안정적으로 실행되어야 한다.
- Jetpack Compose 기반 선언형 디자인 가이드를 충실히 이행해 다양한 비율의 안드로이드 단말기에서도 정상 표시된다.

### 3.5 로컬 저장소
- 시스템은 소비 분석 결과 및 최근 인사이트 정보를 모바일 기기의 Preferences DataStore에 영구 저장한다.
- 홈 화면은 앱이 처음 시작될 때 DataStore에 저장된 데이터를 즉시 읽어와 표시하여, 사용자가 앱을 재실행해도 이전 분석 결과와 소비 인사이트를 별도의 네트워크 요청 없이 즉시 확인할 수 있어야 한다.
