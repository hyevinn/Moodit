# F-04 소비 정보 분석 요청 기능

## 설명

사용자는 당일 누적하여 저장된 다건의 소비 내역 목록을 기반으로 AI 소비 성향 분석을 통합 요청할 수 있다.

## Acceptance Criteria

- [ ] 소비 분석을 요청하려면 최소 1건 이상의 소비 내역이 등록되어 있어야 한다. 등록 내역이 없을 경우 경고 메시지(Toast)가 표시된다.
- [ ] '소비 분석하기' 버튼을 누르면 당일 저장된 모든 소비 내역 정보(카테고리, 금액, 소비 이유, 메모 등)가 분석 엔진 및 결과 화면으로 안전하게 전달된다.
- [ ] 분석 요청 과정에서 `LoadingScreen`으로 라우팅되며, AI 분석 작업을 수행하는 비동기 처리 중임을 알리는 로딩 애니메이션 상태가 표시된다.
- [ ] 비동기 스레드에서 GroqCloud API 및 Llama 3.3 LLM 모델 호출이 시작된다.
