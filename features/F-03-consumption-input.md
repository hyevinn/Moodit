# F-03 소비 기록 및 다건 내역 관리

## 설명

사용자는 직접 금액을 입력하고 카테고리, 소비 이유, 메모를 설정하여 개별 소비 내역을 저장하고 다건의 내역을 등록/수정/삭제하며 누적 관리할 수 있다.

## Acceptance Criteria

- [ ] 사용자는 소비 카테고리(식비, 카페, 교통, 쇼핑, 취미, 자기계발, 기타)를 선택할 수 있다.
- [ ] 사용자는 소비 금액을 텍스트 필드를 통해 숫자로 직접 입력할 수 있으며, 입력 시 숫자 키패드가 나타난다.
  - 금액은 0원을 초과해야 저장 가능하며 최대 999,999,999원까지 입력이 제한된다.
  - 3자리 단위마다 천 단위 구분 쉼표(,)가 실시간 자동 포맷팅되어 노출된다.
- [ ] 사용자는 소비 이유(필요 소비, 자기만족, 기분전환, 유행 영향)를 카드형 버튼 UI를 통해 선택할 수 있다.
  - 카드 내부에 이모지와 텍스트가 정중앙 세로 정렬로 결합되어 노출된다 ("자기만족" -> "🙂", "기분전환" -> "💨", "유행 영향" -> "✨", "필요 소비" -> "✔").
  - 선택된 카드는 강조 색상(primaryContainer)으로 활성화되고, 모든 카드 버튼은 동일한 가로/세로 비율과 크기를 유지한다.
- [ ] 사용자는 오늘 소비 기분 한마디(메모)를 최대 100자까지 선택적으로 입력할 수 있으며 실시간 글자 수가 표시된다.
- [ ] 사용자는 '소비 저장하기' 버튼을 통해 소비 내역을 임시 저장(누적)할 수 있다.
- [ ] 입력 및 저장된 내역들은 '오늘의 소비 내역' 영역에서 `LazyColumn` 리스트 형태로 실시간 출력된다.
  - 리스트 아이템 내부에서 각 카테고리 및 소비 이유에 맞춰 알맞은 이모지가 결합되어 표시된다.
  - 사용자는 리스트 각 아이템 내부의 '수정' 버튼을 클릭해 기저장된 필드 값을 입력 폼으로 불러와 수정 완료할 수 있다.
  - 사용자는 '삭제' 버튼을 클릭해 저장 목록에서 해당 소비 항목을 즉시 제거할 수 있다.
- [ ] 등록된 전체 소비 내역의 금액 합계가 '총 소비 금액' 필드에 실시간 합산 계산 및 갱신되어 포맷팅된 원화 형태로 노출된다.
